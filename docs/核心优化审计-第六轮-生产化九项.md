# 核心优化第六轮：九项生产化增强

本轮针对第五轮之后仍存在的九个生产化缺口继续优化。设计原则是本地默认兼容：不配置安全项时原启动方式不变，不安装可选依赖时自动降级，不要求 Redis、Qdrant 或 Prometheus 服务才能运行。

## 1. 向量检索规模

- 新增可选 `sqlite-vec` KNN 后端，使用 cosine distance 和独立映射表，不再需要在 Python 中扫描全部 JSON 向量。
- `vector_backend` 支持 `auto`、`sqlite-vec`、`json`；`auto` 在扩展不可用时回退。
- embedding 维度变化时清空不兼容旧向量，随后可调用 reindex 回填。
- 使用 `pip install -r requirements-vector.txt` 安装；基础依赖仍可直接运行。

## 2. 多实例后台任务

- `memory_jobs` 增加可用时间、租约 owner 和租约过期时间。
- claim 使用事务原子领取；租约过期后其他 worker 可以安全接管。
- worker 周期续租，旧 owner 无法完成已经被接管的任务。
- 重试使用指数退避；达到上限进入 `failed`，可通过 API/前端重新排队。

## 3. 流式供应商兼容

- 网络错误只在尚未输出 token 时重试，避免重复内容。
- 供应商拒绝 `stream_options` 时自动移除并重试。
- 忽略单条损坏 SSE 数据，识别流内 `error`，保持 tool-call 分片聚合。
- 已经输出正文或工具分片后连接中断会明确失败，不做危险重放。

## 4. 记忆评测

- 种子集扩展为 12 条中文案例，覆盖五类记忆和无需召回的负样本。
- 指标扩展为 Recall@K、Precision@K、MRR、错误注入率、forbidden hit rate、gating accuracy。
- `python -m eval.run_seeded` 可离线运行；加 `--embedding` 使用真实 embedding；`--min-recall` 可作为 CI 阈值。

## 5. 前端记忆运维

- 记忆页新增后台任务表和活动任务轮询。
- failed 任务可手动重试。
- completed 任务通过 shadcn/Base UI AlertDialog 先 dry-run 预览，再确认撤销。
- 撤销结果显示停用数量和恢复旧版本数量。

## 6. 工具权限

- `ToolPolicy` 将 `memorize` 与 `forget_memory` 授权分离，不再用一个布尔值放行全部写工具。
- JSON Schema 校验扩展到 enum、长度、数值范围、数组上限和嵌套对象。
- pre-hook 自身受超时保护；授权决策进入 turn trace metadata。

## 7. API 安全

- 可选 Bearer Token / `X-API-Key`，使用常量时间比较；健康检查保持公开。
- 不安全方法校验 Origin；CORS 来源由 TOML 配置。
- 支持每 IP 每分钟限流和请求体大小限制。
- 前端通过 `VITE_MEMORIA_API_TOKEN` 自动附加 Bearer Token。

## 8. 可观测平台

- `/metrics` 输出 Prometheus 文本，包括请求数与耗时、turn 状态、平均耗时、token 总量、任务状态和有效记忆数。
- API Token 启用后 `/metrics` 同样受保护。
- overview 展示实际向量后端。

## 9. 浏览器自动化

- 引入 Playwright Chromium，使用 API route mock 保持确定性。
- 覆盖“发送流式消息后应用不白屏”和“失败任务重试、撤销预览、确认撤销”。
- 命令：`cd frontend && npm run test:e2e`。

所有新配置都有默认值。旧 `config.toml` 可以继续运行；生产环境再按模板增加 security、worker、retrieval 和 observability 配置。
