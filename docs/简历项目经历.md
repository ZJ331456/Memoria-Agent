# Memoria-Agent 简历项目经历

以下内容与当前仓库已经实现并通过测试的能力一致，可直接粘贴到 LaTeX 简历中：

```latex
\section{项目经历}
\ResumeItem{基于长期记忆与 Tool Calling 的可观测 AI Agent 系统}
[\href{https://github.com/ZJ331456/Memoria-Agent}{\textnormal{项目链接}}]
[2026.04--2026.07]
\textbf{项目描述：}
参考开源 Akashic Agent 的阶段化运行时思想进行独立重构，构建集多轮对话、
跨会话长期记忆、受控工具调用、流式交互、后台记忆任务与可观测运维于一体的
个人智能体系统；后端基于 FastAPI 与 SQLite，前端基于 React/Vite 和 shadcn/ui。

\textbf{核心流程：}
\begin{enumerate}
  \item 设计五阶段 Agent Runtime，串联消息持久化、记忆召回、上下文预算、
  模型推理、Tool Calling 与结果落库，并通过生命周期 Pipeline 解耦运行时模块；
  支持 SSE 增量输出、会话级并发控制、主动取消及断连清理。

  \item 构建 SQLite 长期记忆引擎，融合 FTS5 关键词召回、Embedding 语义检索与
  RRF 排序，实现事实抽取、重复强化、偏好版本替代、来源级撤销和旧版本恢复；
  引入可选 sqlite-vec KNN 索引，并在扩展不可用时自动降级。

  \item 实现带风险分级的 Tool Registry，支持 OpenAI-compatible 结构化工具调用、
  递归 JSON Schema 参数校验、单工具写权限、执行超时、输出裁剪及重复调用熔断，
  降低模型越权写入与工具死循环风险。

  \item 设计基于 SQLite 租约的后台记忆任务机制，通过原子领取、周期续租、
  过期接管、指数退避和失败重试保证多实例执行可靠性，并在前端提供任务状态、
  手动重试、撤销影响预览与确认恢复功能。

  \item 基于 FastAPI、React/Vite 与 shadcn/ui 搭建前后端交互及运维界面，
  提供严格 OpenAPI 契约、Token/Origin/限流安全边界、Prometheus 指标和 Turn Trace；
  建立 27 项后端测试、12 条记忆评测及 Playwright 浏览器回归，记忆评测
  Recall@K 与 MRR 均达到 1.0。
\end{enumerate}
```

当前仓库没有实现 MCP 动态服务器接入、主动信息推送、Telegram 通道或 WebSocket，
因此简历中没有把这些能力写成已完成项。后续真正落地后再补充会更经得起代码追问。
