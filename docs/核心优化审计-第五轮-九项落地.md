# 核心优化第五轮：九项落地说明

> 这是第五轮完成时的历史快照；向量存储等后续状态以《核心优化第六轮：九项生产化增强》为准。

本轮继续提炼 Akashic 的核心运行时经验，但不接入飞书、Telegram、QQ、主动推送等外部能力，也不在运行时引用参考仓库。以下九项均在 Memoria 自身目录内实现。

## 1. 前端依赖隔离

- 删除曾指向外部 `akashic-agent/node_modules` 的污染锁文件和断链软链接，重新生成 `frontend/package-lock.json`。
- `frontend/node_modules` 只由当前项目的锁文件安装，并被 Git 忽略。
- 验收：锁文件无外部路径、外部软链接数为 0、`npm ls --depth=0` 和 `npm run build` 成功。

## 2. 流式响应、取消与断连

- `LLMClient.chat_stream()` 解析 OpenAI-compatible SSE，聚合正文、reasoning、tool-call 分片和 usage。
- `POST /api/sessions/{id}/chat/stream` 将 phase、delta、tool、complete、error、cancelled 事件转为浏览器 SSE。
- 每个 session 只允许一个 active turn；`POST /cancel`、浏览器断连和服务关闭都会取消任务，trace 记录 `cancelled`。
- 前端逐字更新 assistant 气泡，并在生成中显示停止按钮，避免等待期间整屏空白。

## 3. 记忆撤销与恢复

- 自动记忆带 `source_ref`，对应触发该任务的用户消息 ID。
- `POST /api/memories/undo` 支持 `dry_run` 预览；执行后停用该来源创建的记忆，并恢复被它替代的旧版本。
- 恢复使用状态切换而非物理删除，因此审计链仍保留。

## 4. 后台记忆任务与 consolidation

- 主 turn 保存助手消息后只写入 `memory_jobs`，不再同步等待 fast 模型抽取。
- `MemoryJobWorker` 在 FastAPI lifespan 中运行，负责抽取、文本/向量去重、强化和 supersede。
- 任务以 `source_ref` 唯一保证幂等，失败最多尝试 3 次；状态可由 `GET /api/memory-jobs` 查询。

## 5. 检索规划、门控与注入

- `MemoryQueryPlanner` 先用本地规则跳过问候/极短请求，再允许 fast 模型判定是否召回、重写 query、选择 kinds 和 limit。
- 检索采用 SQLite FTS5 关键词候选、内存文本评分、embedding 余弦通道和 RRF 融合。
- 注入按类型设置上限，避免某一类记忆挤占全部上下文；计划、注入量、top score 和 sufficiency 写入 trace metadata。

## 6. 记忆评测集

- `eval/memory_cases.json` 提供偏好、目标、supersede 和负样本基准。
- `python -m eval.memory_eval predictions.json -k 5` 输出 Recall@K、MRR、错误注入率。
- 评测函数是纯函数，便于 CI 使用固定预测做阈值回归，也可接实际检索器生成 predictions。

## 7. 工具安全

- 非只读工具默认拒绝，API 要求 `confirm_write=true`；Agent turn 只有检测到用户明确记住/删除意图才授权记忆写工具。
- JSON Schema 拒绝未知参数和错误类型，数值字段拒绝布尔值。
- 支持全局 pre-hook、每工具超时和最大输出字符数；超长结果截断后再进入模型上下文。
- `CancelledError` 不被包装成普通工具失败，能沿调用链真正终止 turn。

## 8. 可观测性

- trace metadata 保存每次 LLM 调用耗时、重试数、token usage、上下文预算和检索决策。
- 工具 trace 保存名称、参数、状态、耗时和受限预览；敏感键、Bearer 和 `sk-` 字符串继续脱敏。
- 所有 HTTP 响应带 `X-Request-ID` 和 `X-Process-Time-Ms`；overview 增加待处理/失败记忆任务数。

## 9. 数据规模优化

- `memories_fts` 与 `messages_fts` 使用 FTS5 trigram，环境不支持时回退标准 tokenizer。
- 新增、编辑、删除记忆和消息时同步维护索引；删除会话先清理对应消息索引。
- 启动时自动迁移旧表并回填 FTS。向量当前仍以 SQLite JSON 保存，适用于本地中等规模；未来可以在不改变 `MemoryEngine` API 的前提下替换为 sqlite-vec/专用向量库。

